# Session Start Checklist

**Before doing anything else, read `handover.md` in the project root.**

The user maintains this file and updates it between sessions. It contains:
- What was built in the last session
- Critical warnings (like the MoneySmart surname strategy)
- Current pipeline status
- Key contacts and env vars

The user will often say "I have the handover in the preview" — this means `handover.md` is open in the Replit preview pane. Read it immediately with the read tool before asking any questions or doing any work.

**Why:** The user passes this file to each new agent at the start of the day. Ignoring it means asking questions already answered in the file, or worse — undoing work from a previous session.
